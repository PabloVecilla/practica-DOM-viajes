# ✈️ Travel Destinations: Dynamic Landing Page

This project is the second assignment of the Master's program, focusing on the transition from static web design to **Dynamic DOM Manipulation**. The core objective was to render UI components using JavaScript data structures instead of hardcoding them in HTML.

## 🚀 Key Features

### 1. Interactive Banner Rotation
The hero banner features a manual carousel logic. Using an array of local image paths, the site tracks the current image using `getAttribute("src")` and rotates to the next one upon clicking.
* **Logic:** The rotation is dynamic, calculated using `array.length` to ensure it works regardless of the number of images in the folder.

### 2. Automated "Recommended" Section
The recommendation cards are not in the HTML file. They are generated by iterating through an **Array of Objects**.
* Each object contains a title, a description (Lorem Ipsum), and an image URL.
* JavaScript creates an `<article>` element for each entry and injects it into the main container using `appendChild`.

### 3. Dynamic Destination Selector
The travel destination dropdown is built from a simple array of strings. 
* JavaScript iterates through the list of cities.
* It generates `<option>` tags dynamically, converting the values to lowercase for backend-ready data handling while keeping the labels user-friendly.

## 🛠️ Tech Stack
* **HTML5:** Semantic structure for SEO and accessibility.
* **CSS3:** Custom layouts using **Flexbox** and **CSS Grid** for responsive card rendering.
* **Vanilla JavaScript:**
    * DOM Selection (`querySelector`)
    * Event Listeners (`click`)
    * Template Literals for clean HTML injection.
    * Array iteration (`for...of`).
